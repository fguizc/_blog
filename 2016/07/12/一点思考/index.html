    • 对网易前端微专业大作业的总结

    一点思考

    萌新对自己在做网易云前端微专业大作业时的一些总结和思考，记录方便自己更好理解以及规避错误。

    布局 HTML+CSS

    1. 主要是对一些语义化的理解，之前通常只会用 div + ul>li + input + a/p非常基础的标签来写页面布局。 大作业参考了网易教育产品部的原网页发现使用nav/header/footer 更加有效地帮助阅读源码和梳理结构。
    2. 对于ID/CLASS的命名规则真是十分重要，在没有经验直接上手时，往往取名偏向于使命名有意义，但是没有考虑到维护CSS时候需要找到其中一个类名时真的太累，应该由外向内层级递进的命名规则，而且可以结合html标签，例如nav中的类名就可以为hd-navkBox-Link，即为header导航栏中的超链接。
    3. 垂直水平居中的实现，个人喜欢通过transform+translate（）实现，以及flex实现弹性布局，幸好是个人作业不需要考虑太多向下兼容，广泛支持的新特性使用真是让人十分愉快。
    4. 不足的是没有尽量多地贴合响应式布局，按照百分比设置长高会出现莫名其妙的布局bug。

    Javascript

    1. 组件的思想真的太重要，就拿轮播组件来说，在网页中应该有两处使用，但是由于个人能力以及嫌麻烦，并没有进行封装，只是简单些了个逻辑，导致意识到需要第二次调用轮播组件时候整个人都是懵逼状态。研究了下组件，觉得以后很有必要按照 需求->原型->抽象->组件 这样的思路来写，凡事动工前应该考虑清楚，不仅仅是功能需求上，还有一些意外情况都应该在内。
    2. DOM基础知识在写作业中就暴露出很多问题，更多的是对于一些操作不熟练，表现在最根本的节点插入,甚至一些语法都忘记（querySelector/All 这一项就完全没有印象，恰恰忘记了一个十分强大的原生DOM操作）。不得不得边写的时候停下来google语法，效率相当低，总结起来就是实操不够，学习的时候理解也不够深入。

    补充

    1. 其实这次大作业让我看到的问题不仅仅是上面，最重要的是认识到 如何思考书写浏览器能够识别的代码 。在实现一些样式或者交互的过程中，发现很多东西并不是眼见为实，例如很多时候CSS是通过层叠让呈现效果看起来是一体，设置边框为周围的颜色或者设置透明度，通过借用一个1px 100%的无内容填充实现布局的居中，图片的淡入淡出通过控制透明度，图片的轮播通过改变图片的位置，轮播中自动翻页是靠绑定一个点击事件，在没有接触到前端时很多看起理所当然的事情，在现在都发生了改变，欺骗了眼睛但是却让人感觉效果流畅，整体一致。这种思考方面帮我解决了一些最开始接触完全没有方向的问题。
