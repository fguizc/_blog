    布局·HTML标签语义化

    什么是html标签语义化

    web语义化是指通过HTML标记表示页面包含的信息，包含了HTML标签的语义化和css命名的语义化。 HTML标签的语义化是指：通过使用包含语义的标签（如h1-h6）恰当地表示文档结构 css命名的语义化是指：为html标签添加有意义的class，id补充未表达的语义。

    为什么要使用语义化标签

    • 去掉样式后页面呈现清晰的结构
    • 盲人使用读屏器更好地阅读
    • 搜索引擎更好地理解页面，有利于收录
    • 便团队项目的可持续运作及维护

    布局·元素类型

    HTML元素可以分为两类

    • 块级元素(block level element)
    • 行级元素(inline element)

    两者区别

    1. 块级元素会独占一行(即某一块级元素后紧跟的元素会另起一行布局)
    2. 块级元素可以设置width/height 属性,而行级元素设置无效(tip:margin:0 auto;对块级元素生效)
    3. 块级元素 width normal=100%,而行级元素根据自身内容或者子元素决定其宽度

    让我们来举个栗子

    若既想让元素在行内显示，又能设置宽高，可以设置：

    display:inline-block

    此外 设置元素的position(absolute|fixed)/float也会将元素转换为block.

    盒模型

    页面上每一个元素(!包含 行内元素)都可以看做一个盒子,即盒模型(box model).

    Margin值的继承

    1. margin值的增加不会影响背景颜色的尺寸
    2. margin传递与叠加

    子元素的上下margin会传递给父级，但父级如果有边框就会阻止子级margin传递，父级是BFC也会阻止margin的传递。

    BFC(Block formatting contexts)

    什么是BFC

    在一个Web页面的CSS渲染中，块级格式化上下文 (Block Fromatting Context)是按照块级盒子布局的

    浮动元素和绝对定位元素，非块级盒子的块级容器（例如 inline-blocks, table-cells, 和 table-captions），以及overflow值不为“visiable”的块级盒子，都会为他们的内容创建新的BFC（块级格式上下文）。

    一个HTML元素要创建BFC，则满足下列的任意一个或多个条件即可：

    1. float value != none
    2. position value !=static|relative
    3. display value === inline-block|table-cell|flex|table-caption|inline-flex
    4. overflow !=visiable

    BFC可以干什么

    1. 清除浮动
    2. 解决外边距折叠

    Position property

    position 属性值

    • static (Defaults)
    • relative (相对定位)
    • absolute (绝对定位)
    • fixed (绝对定位-根据浏览器窗口定位)

    1. relative 元素在文档流中占有一个位置,相对定位就是将元素偏离元素的默认位置,但是在文档流中依然保持着原有位置,并未脱离文档流,即便对元素设置偏移量,元素的下一个兄弟元素位置也不受影响而补位. 如图所示,设置相对偏移量后,DIV .C 依旧没有补位

    2. absolute 设置position:absolute的元素会根据节点树中的父(祖)来进行定位,若节点树往上查找均无设置position:relative的元素,则绝对定位会根据/来进行定位

    3. fixed fixed可视为一种特殊的absolute,但其与absolute最大的不同在于 absolute的"根元素"可以设置,而fixed的"根元素"始终是浏览器的窗口

    ( ! tip:absolute|fixed均会脱离文档流)

    position设置带来的后果

    1. 块状元素在position(relative/static)的情况下width为100%，但是设置了position: absolute之后，会将width变成auto（会受到父元素的宽度影响）。
    2. 应用了position: relative/absolute的元素，margin属性仍然有效，以position:relative来举例。如果设置了left、top、bottom、right的属性 。
    3. position: absolute忽略根元素的padding。
    4. 在IE6/7中设置position属性后会导致z-index属性失效
    5. 行内元素在应用了position：absolute之后会改变display。
    6. 应用了position: absolute / relative之后，会覆盖其他非定位元素（即position为static的元素），如果你不想覆盖到其他元素，也可以将z-index设置成-1。

    参考自CoffeeDeveloper的博客

    Float

    float 属性值

    • right
    • left
    • none
    • inherit

    float 要点

    1. 只能左右浮动没有上下浮动
    2. 元素设置float后,会 脱离文档流 (和absolute一样) ,不再占据原有空间位置
    3. 浮动不会作用该元素的上一个兄弟元素
    4. 浮动后,该元素的下一个兄弟元素会补齐浮动元素原有位置
    5. 浮动后,该元素会变为 块级元素
    6. 清除浮动

    清除浮动

    方法一

    .clearfix:after {
	 content: ".";
	 display: block;
	 height:0;
	 clear: both;
	 visibility:hidden;
}
.clearfix{*+height:1%;}

    !注意此方法会改变原html的结构会产生:after,当最为常用

    方法二:

    .clearfix{overflow:auto;height:1%;}

    ##布局——水平垂直居中

    水平/垂直居中可以分为两类

    1. 块级元素的水平垂直/居中(如div)
    2. 行级元素的水平垂直/居中(如a/img/span)

    1. absolute + transform

      .parent{position:relative;}
.child{position:absolute;
       top:50%;
       left:50%;
       transform:translate(-50%,-50%);}

    2. inline-block+text-align+tabel-cell+vertical-align

      .parent{text-align:center;
		display:tabel-cell;
		vertical-align:middle;}
.child{display:inline-block;}

    3. flex+justify-content+align-items

      .parent{display:flex;
		justify-content:center;
		align-items:center;}
