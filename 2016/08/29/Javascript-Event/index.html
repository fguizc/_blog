    Javascript Event

    事件的触发

    在JS中，当用户进行操作，如 点击按钮，鼠标滑动，鼠标点击，鼠标松开，文本框获得、失去焦点等，就是事件的触发。

    事件实现模式

    1. 内联模式
    2. 脚本模式

    内联模式

    直接在HTML标签中添加事件，这种模型是最传统简单的处理事件的方法，
但此模式中将HTML与事件处理混写，没有行为与结构样式分离。
如：
	<input type = "button" value = "按钮" onclick = "alert('hello world');">
或
	<input type = "button" value = "按钮" onclick = "btnClick();" >

    内联模式调用函数，函数不能放入window.onload内

    脚本模式

    内联模式不符合“行为与结构样式分离”的原则，脚本模式则是将事件的处理写入JS文件中，使用脚本模式首先要获得元素节点对象，
所以JS代码需要在HTML结构和样式渲染完成后再执行（这也是一直提倡的）
如：
	var btnClick = document.querySelector(Element);
	btnClick.onclick = function (){};
或
	var btnClick = document.querySelector(Element);
	btnClick.onclick = fn;
	function fn(){};

    事件处理类型

    • 鼠标事件
    • 键盘事件
    • HTML事件

    鼠标事件

    • click :当点击鼠标按钮并松开时触发
    • dblclick ：当双击鼠标按钮时触发
    • mousedown：当按下了鼠标还未松开时触发
    • mouseup ：释放鼠标按钮时触发
    • mouseover：当鼠标移入某个元素的那一刻触发
    • mouseout ：当鼠标移出某个元素时触发
    • mousemove：当鼠标在某个元素上移动时

    键盘事件

    • keydown ：用户按下键盘上某个键触发，如果按住不放，会不断重复
    • keypress ：用户按下键盘上的字符键触发，如果按住不放，会重复
    • keyup ：用户释放键盘上的某个键触发

    HTML事件

    • load ：当页面加载完全后触发
    • unload ：当页面完全卸载后触发
    • select ：用户选择文本框（input/textarea）中的内容触发
    • change ：当文本框(input/textarea)内容改变且失去焦点触发
    • focus ：当页面或者元素获得焦点
    • blur ：当页面或元素失去焦点时触发
    • submit ：当用户点击提交按钮在<form>元素节点上触发
    • reset ：当用户点击重置按钮在<form>元素节点上触发
    • scroll ：当用户滚动滚动条的元素时触发

    事件属性

    • 鼠标事件(常用来配合鼠标触发一些特殊事件)

      1. shiftKey 判断是否按下shift键
      2. ctrlKey 判断是否按下ctrl键
      3. altKey 判断是否按下alt键
      4. metaKey 判断是否按下Meta键(Windows|Cmd)

    • 键盘事件

      1. keyCode 当keydown或keyup时，按键会对应一个代码，keyCode属性的值与ASCII码中大写字母或数字的编码相同(按键不包括亮度、音量…的按键)
      2. charCode 该属性只有在keypress事件时才包含值，这个值是按下的键所代表字符的ASCII码，此时的keyCode可能为0也可能为所按键的编码

    事件触发的流程

    描述的是页面接受事件的顺序，当多个具有事件的元素堆叠在一起，触发其中的元素，会引发所有层叠在触发事件范围内的事件都被触发。事件流包括两种：冒泡、捕捉(大多数浏览器默认行为为 冒泡)。

    事件的冒泡

    冒泡的行为

    事件冒泡是指，在页面上层 节点触发的事件会继续传递给下层节点，这种传递方式称之为冒泡传递。

    阻止事件冒泡

    在多数情况，我们只想让指定的节点触发事件，而不发生传递关系，需要阻止冒泡

    1. 取消冒泡 oEvent.cancelBubble = true;
    2. 停止传播 oEvent.stopPropagation();

    事件的监听

    事件的监听是对JS中一种事件触发机制，可以通过添加事件监听器的方式赋予元素事件及执行函数

    添加事件监听器

    target.addEventListener(type,listener[,useCapture]);
type:表示所监听 事件类型 的一个字符串。
listener:当指定的事件类型触发后被通知到的一个对象，该参数必须是实现EventListener 接口的一个 对象或函数
useCapture：是一个可选值，value = true，执行捕获，value = false，执行冒泡,若没有指定，默认为false

    移除事件监听器

    target.removeEventListener(type,listener);

    为什么使用事件监听器

    addEventListener 是W3C DOM中提供的注册事件监听器的方法。它的优点包括：

    1. 它允许给一个事件注册多个监听器。
    2. 它提供了一种更精细的手段控制事件监听器的触发阶段。（即可以选择捕获或者冒泡）。
    3. 它对任何DOM元素都是有效的，而不仅仅只对HTML元素有效。

    多个相同的事件处理器

    如果多个相同的 EventListener被注册到同一个EventTarget，而且参数都是相同的，那么重复的实例会被抛弃。所以这么做不会使得 EventListener 被调用两次，也不需要用removeEventListener手动清除多余的EventListener ，因为重复的都被自动抛弃了。

    1.实现阻止右键以及自定义右键

    阻止右键

    通过返回一个false来阻止

    document.oncontextmenu = function(){
	alert('右键被按下');
	return false;	
}

    自定义右键

    首先阻止系统默认右键 通过判定鼠标事件中触发的右键来显示一个自定义列表达到“伪装右键”效果

    2.超链接跳转的二次确定

    target.onclick = function () {
	if(confirm('确定离开吗?')){
		return true;
	}else{
		return false;
	}
}
